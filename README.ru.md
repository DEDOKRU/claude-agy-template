# claude-agy-template

[English](README.md) | [Deutsch](README.de.md) | **Русский**

Шаблон проекта, связывающий **Claude Code** (архитектор и ревьюер) с **Antigravity CLI** (`agy`, исполнитель) для экономии токенов Claude: Claude пишет контракт и проверяет результат; agy пишет код и гоняет тесты; PowerShell-скрипт связывает их; git-ветка служит предохранителем.

```text
Claude Code  -> /agy-handoff   -> контракт (spec + критерии приёмки + промпт)
Claude Code  -> /agy-implement -> tools/invoke-antigravity.ps1 -> agy пишет код и тестирует
Claude Code  -> перезапускает проверку сам + читает diff -> ACCEPT / NEEDS_FIXES / REJECT
ты           -> commit / merge вручную
```

Почему это экономит токены: Claude не пишет boilerplate, не нянчит прогоны тестов и не перечитывает весь проект. Дорогая работа (реализация, отладка красных тестов, повторные прогоны) крутится на стороне Gemini через Antigravity.

## Требования

- Windows с PowerShell 5.1+ (мост и установщик — PowerShell-скрипты)
- CLI [Claude Code](https://code.claude.com)
- [Antigravity CLI](https://antigravity.google) — `agy` в PATH, один раз залогинен интерактивно
- git

## Установка — один и тот же блок для нового и существующего проекта

Открой PowerShell **в папке проекта** и вставь этот блок как есть:

```powershell
$t = "$env:TEMP\agy-tpl"
if (Test-Path $t) { Remove-Item -Recurse -Force $t }
git clone --depth 1 https://github.com/DEDOKRU/claude-agy-template.git $t
powershell -ExecutionPolicy Bypass -File $t\tools\install-into-project.ps1 -Target .
Remove-Item -Recurse -Force $t
```

Установщик идемпотентен (можно перезапускать) и никогда не перезаписывает твои файлы:

- копирует скрипт-мост, команды `/agy-handoff` + `/agy-implement` и шаблоны handoff;
- **дописывает** секцию правил в существующий `CLAUDE.md` (или создаёт его);
- **дописывает** `.agent_handoff/**/logs/` в `.gitignore`;
- регистрирует путь проекта в `trustedWorkspaces` agy (без этого headless-запуски виснут).

### Сценарий 1: новый проект с нуля

1. Кнопка **Use this template** на GitHub → создай свой репозиторий.
2. Склонируй его локально через `git clone`.
3. Выполни блок установки в папке — файлы будут пропущены (уже есть), но workspace зарегистрируется у agy.

### Сценарий 2: существующий проект

1. Убедись, что это git-репозиторий хотя бы с одним коммитом (`git init` + первый коммит, если нет — механика checkpoint/rollback/diff требует базовой точки).
2. Выполни блок установки, проверь `git status`, закоммить.
3. Перезапусти Claude Code в проекте, чтобы подхватились новые команды.

### Сценарий 3: обновить уже подключённый проект

Установщик намеренно не перезаписывает существующие файлы, поэтому обновление — через удаление:

```powershell
Remove-Item tools\invoke-antigravity.ps1, .claude\commands\agy-handoff.md, .claude\commands\agy-implement.md
```

Затем снова блок установки — он положит свежие копии.

## Рабочий цикл

1. `/agy-handoff <описание задачи>` — Claude создаёт рабочую ветку `agy/<имя>` и контракт: что делать, какие файлы можно трогать, как проверять, критерии приёмки.
2. `/agy-implement` — мост делает checkpoint-коммит, agy реализует и проверяет (обычно 5–30 минут), затем Claude ревьюит: сам перезапускает команды проверки, сверяет diff со списком разрешённых файлов, проходит по критериям.
3. Вердикт:
   - **ACCEPT** — ты мержишь (или открываешь PR).
   - **NEEDS_FIXES** — замечания уже в `REVIEW_NOTES.md`; запускаешь `/agy-implement continue`, agy правит в той же сессии, Claude ревьюит заново.
   - **REJECT** — откат `git reset --hard <checkpoint>` (хэш печатает скрипт) и новый handoff.

Единственные ручные действия во всём цикле — две команды выше и финальный merge: Claude никогда не коммитит, не мержит и не пушит — это предохранитель.

## Предохранители

- Мост отказывается работать на `main`/`master` и без handoff-файлов.
- Checkpoint-коммит перед каждым запуском agy: `git diff HEAD` показывает ровно правки агента; откат — одна команда.
- agy запускается с `--sandbox`; `--dangerously-skip-permissions` принимается только вместе с sandbox.
- `--print-timeout 30m` (дефолтные 5 минут agy обрезали бы реальную задачу).
- Отчёты agy — недоверенный вход: Claude обязан сам перезапустить команды проверки перед ACCEPT.
- Команды проверки подбираются под тип задачи: тест-сьют для долгоживущего кода, обычный запуск + sanity-критерии для одноразовых research-скриптов.

## Гигиена токенов (твои привычки — шаблон это не автоматизирует)

- **`/clear` после каждой завершённой задачи.** Одна бесконечная сессия на 20 задач — главный пожиратель лимита: устаревший контекст оплачивается в каждом сообщении. Если понадобится вернуться — сначала `/rename <имя>`.
- **Исправлял Claude дважды — не исправляй третий раз.** Контекст уже загрязнён; `/clear` и более точный промпт дешевле и работают лучше.
- **`/context` и `/mcp` периодически.** Неиспользуемые MCP-серверы стоят контекста в каждом сообщении — отключай ненужные для текущей работы.
- **`/compact` с инструкцией, а не голый**: `/compact сохрани только активную задачу, изменённые файлы, решения, команду проверки и следующий шаг` (то же правило вшито в CLAUDE.md проекта — авто-компакт его учитывает).
- **Для больших проектов заведи `PROJECT_MAP.md`** в корне (шаблон в `.agent_handoff/templates/`) — Claude читает карту вместо обхода дерева.
- **Прерванная работа продолжается по файлу, а не по памяти чата**: правила проекта требуют вести `.agent_handoff/current/SESSION_STATE.md` при многосессионных задачах — новая сессия стартует с него, не восстанавливая контекст за токены.

## Грабли, найденные на живых прогонах (скрипты уже их учитывают)

- agy в `-p` print-режиме ждёт EOF на stdin и висит вечно на открытом пайпе — мост закрывает stdin через `$null |`.
- Workspace, отсутствующий в `trustedWorkspaces` (`~/.gemini/antigravity-cli/settings.json`), вешает headless-запуски — установщик регистрирует его.
- agy молча самообновляется при старте (загрузка выглядит как зависание — это разовая пауза).
- Дефолтный print-timeout agy — всего 5 минут; мост ставит 30.
- `agy models` печатает список только в настоящем интерактивном терминале.
- PowerShell 5.1 читает скрипты без BOM в ANSI-кодировке: длинное тире в UTF-8 превращается в «умную кавычку», ломающую парсинг строк — поэтому оба скрипта чистый ASCII.

## Структура репозитория

```text
.claude/commands/agy-handoff.md    # Claude: создать контракт для agy
.claude/commands/agy-implement.md  # Claude: запустить agy и отревьюить
tools/invoke-antigravity.ps1       # мост: проверки, checkpoint, запуск agy, сводка
tools/install-into-project.ps1     # установщик для новых/существующих проектов
.agent_handoff/templates/          # шаблоны контракта, session-state и карты проекта
CLAUDE.md                          # правила для Claude в этом репо
```
